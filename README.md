## 🚀 About

SQL interviews are not about memorizing queries.

They are about recognizing **patterns**.

This repository is a collection of SQL design patterns that appear repeatedly in real interviews at companies like:

- Uber
- Google
- Amazon
- Microsoft
- PhonePe
- Razorpay
- Swiggy
- Zomato
- Meesho
- Atlassian

Every pattern contains:

- 📌 Business Requirement
- 📌 Pattern Recognition
- 📌 SQL Approach
- 📌 Interview Thinking
- 📌 Real Business Use Cases
- 📌 SQL Solution
- 📌 Multiple Practice Questions

The objective of this repository is to master SQL by learning **patterns instead of memorizing questions**.

1. Business Requirement

3. SQL Pattern
4. SQL Functions
5. Final Query
6. Interview Explanation
